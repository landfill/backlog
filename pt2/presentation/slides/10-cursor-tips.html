Cursor 실무: 컨텍스트 관리법
비개발자도 바로 쓸 수 있는 컨텍스트 관리
컨텍스트 채우는 방법
@Cursor Rules
매 대화에 자동 주입되는 상시 규칙. 프로젝트 규칙, 내 업무 방식을 여기에 정의
@Past Chats
이전 대화 참조. 맥락을 이어가고 싶을 때 과거 결정사항 불러오기
@파일 / @폴더
내가 원하는 것만 골라서 주입. 불필요한 것은 넣지 않아야 컨텍스트 절약
언제 새 에이전트를 열어야 하나
1
Cursor 하단 컨텍스트 바 토큰 사용량이 절반을 넘었을 때
2
주제가 바뀔 때 — 기능 A 완료 후 기능 B 시작
3
AI가 같은 실수를 반복할 때 → 컨텍스트 오염 신호. 더 대화해도 나아지지 않습니다
📁 비개발자를 위한 폴더 구조 팁
_context/ 폴더를 만들고 회의록·요구사항·결정 로그를 쌓으면
새 에이전트에 @_context 하나로 모든 배경을 즉시 주입
_context/ ├─ meeting_notes.md ├─ requirements.md └─ decision_log.md
