### 컨텍스트 윈도우 안에 들어가는 것들

**항상 들어가는 것**
- ⚙️ **시스템 프롬프트** — 도구가 AI에게 주는 초기 지시
- 💬 **내 질문 (전체 누적)** — 이번 대화의 모든 입력
- 🤖 **AI 답변 (전체 누적)** — AI가 생성한 모든 응답

**추가될 수 있는 것**
- 📎 **첨부 파일** — 올린 문서, PDF, 엑셀
- 🖼️ **캡처 이미지** — 붙여넣은 스크린샷
- 🔧 **MCP 툴 결과** — 외부 도구 호출 및 응답값

> ⚠️ **핵심:** 모든 항목이 동일한 공간을 공유합니다.
> 파일이 크면 대화 가능한 공간이 줄어듭니다.

## SECTION 03 — 프롬프트 → 컨텍스트 엔지니어링으로 `3분`
> 질문법이 아닌 맥락 관리가 핵심입니다

### 패러다임 전환

```
과거: 프롬프트 엔지니어링  →  어떻게 질문할까?
지금: 컨텍스트 엔지니어링  →  무엇을 어떻게 채워넣을까?
```

---

### 전달 방식이 결과를 바꾼다

같은 AI에게 같은 일을 시켜도, **무엇을 함께 전달하느냐**에 따라 결과가 달라집니다.

| | 프롬프트만 | 컨텍스트 포함 |
|---|---|---|
| **입력** | "환불 정책 만들어줘" | "월 9,900원 구독형인데, 가입-환불 반복 악용이 30%야. 지난번에 7일 이내 전액 환불로 바꿨더니 1일 사용 후 환불 패턴이 늘었어. 그래서 결제 직후 24시간 이내만 허용하는 방향으로 판단했어." |
| **결과** | 어디서든 쓸 수 있는 일반 템플릿 | **우리 상황에 맞는** 정책안 |

단순히 현황 정보가 아닙니다.
이전에 **어떤 시도를 했고**, **어떤 결과가 있었고**, **그래서 어떤 판단을 내렸는지** — 이 맥락이 차이를 만듭니다.

> **AI는 받은 맥락을 증폭하는 도구입니다.**
> 좋은 맥락을 넣으면 좋은 결과가 나오고,
> 잘못된 맥락을 넣으면 틀린 결과를 그럴듯하게 포장합니다.

---

### 💡 컨텍스트가 충분한지 확인하는 법

> AI에게 줄 모든 정보를 인쇄해서 **옆 동료에게 건네보세요.**
> 그 동료가 요청한 일을 못 하면, AI도 못 합니다.
> AI는 마음을 읽지 못합니다. 당신에게 당연한 것을 **명시적으로** 전달해야 합니다.

---

### 컨텍스트가 가득 찰 때 3가지 선택

| 방법 | 행동 | 결과 |
|---|---|---|
| **Clear** | 새 대화창 열기 | 깨끗하게 리셋. 하지만 이전 맥락 전부 소멸 |
| **Compact** | Claude Code `/compact` | 대화를 요약 압축. 핵심 맥락은 유지 |
| **새 에이전트** | Cursor 새 Agent 창 | 깨끗하지만 역시 이전 기억 없음 |

> ❗ **핵심 문제:** 어떤 방식이든 새로 시작한 AI는 기억이 없습니다.
> 바이브코딩 세션에서 2시간 쌓아온 결정, 실패한 시도, 설계 이유가 전부 날아갑니다.

## SECTION 04 — 하니스(Harness): 맥락을 파일로 영속화 `2.5분` 🖼️
> 기억은 압축되고, 채팅창은 휘발됩니다. 파일만 남습니다

### 기억만으로는 부족한 이유

"머릿속에 있는 걸 AI한테 말로 전달하면 되지 않나?"라고 생각할 수 있습니다.

하지만 사람의 기억에는 구조적 한계가 있습니다.
경험을 저장할 때 **자세한 맥락은 버리고 결론만 압축**해서 기억합니다.

| 기억나는 것 | 기억 안 나는 것 |
|---|---|
| "그때 A로 결정했다" | 어떤 대안을 검토했는지 |
| | 왜 B가 아니라 A였는지 |
| | 그 판단의 근거가 뭐였는지 |

기억이 안 나면 AI에게 전달할 수 없고,
전달할 수 없으면 **교과서 답만** 돌아옵니다.

> 🗣 **강사 노트**
> "그때 왜 그렇게 결정했어요?" — 기억 안 남
> "비슷한 거 전에 했었는데..." — 기억 안 남
> "퇴사한 동료 업무 인수" — 지식 전부 증발
> 청중의 고개가 끄덕여지면 다음으로 넘어갑니다.

그래서 지식이 머릿속이 아닌 **별도 공간에 기록**되어 있어야 합니다.

---

### 파일이 기억한다

바이브코딩 커뮤니티가 발전시킨 해결책입니다.
대화는 휘발되지만 파일은 남습니다.
새 에이전트가 시작될 때 이 파일을 읽으면 즉시 맥락을 파악합니다.

| 파일 | 위치 | 역할 |
|---|---|---|
| `CLAUDE.md` | 프로젝트 루트 | Claude Code가 매 세션 자동으로 읽는 기억 |
| `Cursor Rules` | `.cursorrules` | Cursor 에이전트에 항상 자동 주입되는 규칙 |
| **`SKILL.md`** | 어디든 | 특정 업무를 위한 맥락 + 지시 패키지 → `@파일`로 주입 |

> "스킬 파일을 쓴다는 것
> = 파일 안의 텍스트를 새 에이전트의 컨텍스트 윈도우에 **주입**하는 것
> = AI에게 매번 온보딩 시키는 대신, 업무 매뉴얼을 건네는 것"

## SECTION 05 — Cursor 실무 팁 `2분`
> 비개발자도 바로 쓸 수 있는 컨텍스트 관리 루틴

### 컨텍스트 채우는 방법

| 명령 | 설명 |
|---|---|
| `@Cursor Rules` | 매 대화에 자동 주입되는 상시 규칙. 프로젝트 규칙, 내 업무 방식을 여기에 정의 |
| `@Past Chats` | 이전 대화 참조. 맥락을 이어가고 싶을 때 과거 결정사항 불러오기 |
| `@파일` / `@폴더` | 내가 원하는 것만 골라서 주입. 불필요한 것은 넣지 않아야 컨텍스트 절약 |

---

### 언제 새 에이전트를 열어야 하나

| 신호 | 설명 |
|---|---|
| 신호 1 | Cursor 하단 컨텍스트 바 토큰 사용량이 **절반을 넘었을 때** |
| 신호 2 | **주제가 바뀔 때** — 기능 A 완료 후 기능 B 시작, 문서 완성 후 다음 문서 |
| **신호 3** | **AI가 같은 실수를 반복할 때** → 컨텍스트 오염 신호. 더 대화해도 나아지지 않습니다 |

> 🗣 **강사 노트**
> 실제 Cursor 화면을 열어서 하단 컨텍스트 바를 보여줍니다.
> "이 숫자가 절반을 넘으면 새 에이전트를 여세요"라고 직접 짚어줍니다.

---

### 비개발자를 위한 폴더 구조 팁

`_context/` 폴더 하나를 프로젝트 안에 만드세요.
회의록, 요구사항, 결정 로그를 텍스트 파일로 쌓아두면
`@폴더`로 통째로 주입할 수 있습니다.

```
_context/
├── meeting_notes.md   # 회의에서 결정된 내용
├── requirements.md    # 기능 요구사항, 제약 조건
└── decision_log.md    # 왜 이렇게 결정했는지 이유
```

→ 새 에이전트가 시작되면 `@_context` 하나로 모든 배경을 즉시 주입할 수 있습니다.

## SECTION 08 — 클로징 `1분`

```
AI가 성과를 평준화하고 있습니다.
하지만 차이를 만드는 건 프롬프트가 아니라 컨텍스트,
그리고 그 컨텍스트는 나의 지식에서 나옵니다.

기억은 압축됩니다. 기록만이 남습니다.
컨텍스트를 관리하는 사람이 AI를 제대로 쓰는 사람입니다.

같이 달리다 보면, 비로소 보입니다.
```

---

### 📌 오늘 배운 것 7가지

| | |
|---|---|
| 1 | AI가 성과를 평준화하고 있다. 하지만 **고유한 판단이 필요한 영역에서는 오히려 역효과**. 전문성의 가치는 사라지지 않았다. |
| 2 | 차이를 만드는 건 프롬프트 기술이 아니라, **내 경험과 판단이 담긴 맥락(컨텍스트)**이다. |
| 3 | AI는 받은 맥락을 증폭하는 도구다. 좋은 맥락 → 좋은 결과, 잘못된 맥락 → 그럴듯한 오답. |
| 4 | 컨텍스트 윈도우 = AI의 작업 메모리. 가득 차면 AI 품질이 저하된다. |
| 5 | **기억은 압축된다.** 결론만 남고 맥락은 사라진다. 기록이 있어야 AI에게 전달할 수 있다. |
| 6 | 새 에이전트는 기억이 없다. 그래서 하니스(파일)가 필요하다. |
| **7** | **스킬 파일은 코드가 아닙니다. 누구든 텍스트로 만들어 쓸 수 있습니다.** |
