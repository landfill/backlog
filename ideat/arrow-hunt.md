# Arrow Hunt 3D — PRD (Product Requirements Document)
**버전** 0.1.0 | **작성일** 2026-03-21 | **작성자** 병광

---

## 1. 제품 개요

### 1.1 제품 비전
스마트폰 한 대와 맨손만으로 즐기는 1인칭 3D 양궁 게임.
별도 컨트롤러 없이 실제 활을 쏘는 것과 유사한 신체 제스처(주먹 쥐기 → 당기기 → 펼치기)를
카메라 기반 핸드 트래킹으로 인식하여, 3D 공간에 배치된 타겟을 조준·적중하는 경험을 제공한다.

### 1.2 핵심 가치 제안
- **Zero Hardware** : 웹 브라우저 + 전면/후면 카메라만으로 즉시 플레이
- **Body-as-Controller** : 실제 양궁 동작을 모사한 직관적인 제스처 인터랙션
- **3D Immersion** : Three.js 기반 원근감 있는 1인칭 시점, 물리 기반 화살 궤도
- **Zero Install** : 단일 HTML 파일 배포, CDN 의존성만으로 구동

### 1.3 타겟 사용자
| 세그먼트 | 설명 |
|----------|------|
| 캐주얼 게이머 | 가볍게 시도하는 10-30대, 모바일 우선 |
| AR/XR 체험 관심층 | 핸드 트래킹·몸 인터랙션에 흥미 있는 기술 얼리어답터 |
| 교육/전시 활용처 | 핸드 트래킹 기술 데모가 필요한 기업·기관 |

## 3. 기능 요구사항

### 3.1 핵심 기능 (Must Have)

#### F-01. 핸드 트래킹 기반 제스처 인식
- **사용 기술** : MediaPipe Hands (웹 버전, CDN)
- **감지 손** : 단일 손 (maxNumHands: 1)
- **모델 복잡도** : 1 (정확도·성능 균형)
- **제스처 정의**

| 제스처 | 감지 조건 | 게임 액션 |
|--------|-----------|-----------|
| 주먹 (Fist) | 4개 이상 손가락 tip/wrist 거리비 < 0.80, 연속 4프레임 | 시위 잡기·당기기 시작 |
| 손 펼침 (Open) | 3개 이상 손가락 tip/wrist 거리비 > 1.05, 연속 3프레임 | 시위 놓기 → 발사 |
| 손 없음 | 랜드마크 미검출 | 당기는 중이면 강제 발사 |

- **오인식 방지** : 히스테리시스 카운터 (연속 N프레임 조건) 적용
- **엄지 별도 처리** : tip↔index_MCP 거리 기준으로 교차 여부 판단

#### F-02. 충전(Draw) 시스템
- 주먹 인식 시작부터 `DRAW_MS` (기본 1,600ms) 만에 100% 충전
- `drawCharge` (0.0 ~ 1.0) 값으로 이하 항목 제어
  - 화살 발사 속도 : `22 + charge * 30` (m/s 환산)
  - 시위 당김 깊이 : bowString Z축 오프셋
  - 조준경 크기 및 arc 표시
  - HUD 충전 바

#### F-03. 3D 씬 및 1인칭 카메라
- **엔진** : Three.js r128 (CDN)
- **씬 구성**
  - 스카이 셰이더 (GLSL, 수평선 그라데이션)
  - 잔디 Ground Plane (300×300)
  - 비포장 Path (타겟 방향으로 연장)
  - 프로시저럴 침엽수 (ConeGeometry 레이어드, 좌우 10그루 이상)
  - 안개 (FogExp2, 밀도 0.011)
- **카메라** : PerspectiveCamera (FOV 72), 시점 고정 Y=1.65m

#### F-04. 조준 방식 (듀얼 입력)

| 입력 | 구현 방식 | 특이사항 |
|------|-----------|----------|
| 모바일 자이로 | deviceorientation (gamma/beta) → YXZ 오일러 | iOS Safari는 requestPermission 필요 |
| 모바일 터치 드래그 | touchmove delta → tYaw/tPitch | 자이로 미지원 폴백 |
| PC 마우스 (locked) | Pointer Lock API movementX/Y | 클릭 시 활성화 |
| PC 마우스 (free) | 뷰포트 상대 좌표 | Pointer Lock 미획득 시 |

- 카메라 smooth damp : `camYaw += (tYaw - camYaw) * 0.12` (매 프레임)
- Pitch 범위 : -1.1 ~ +0.65 rad 클램프

#### F-05. 화살 발사 및 물리
- **발사 시점** : 손 펼침 감지 시 `shootArrow(drawCharge)` 호출
- **초기 속도** : 카메라 Forward 방향 벡터 × speed
- **중력** : `vel.y -= 9.8 * dt` (매 프레임)
- **방향 업데이트** : 속도 벡터로 LookAt 회전 (화살 탄도 회전 표현)
- **시각 요소**
  - 샤프트 : CylinderGeometry (R=0.022, L=1.6), Phong 재질
  - 촉 : ConeGeometry (R=0.044, L=0.22), emissive 발광
  - 깃털 (fletching) : PlaneGeometry × 3, 120° 배치
  - 트레일 : LineBasicMaterial, 25포인트 잔상, 발광 노란색
  - 발사 플래시 : 화면 전체 radial gradient 오버레이, 60ms 유지 후 0.4s fade
  - PointLight : 촉 위치에 orange 점 광원 (intensity 0.8, distance 2.8)

#### F-06. 타겟 시스템
- **기본 배치** : 5개 (거리 12m, 20m, 28m, 36m, 44m)
- **타겟 구성** : 5링 과녁 (흰/검/파/빨/노), 나무 판자 + 다리
- **크기** : 거리 비례 증가 (r = 0.52 + |z| * 0.004), 원근감 반영
- **이동 타겟** : 일부(~40%)는 좌우로 왕복 이동 (vx = ±0.016/frame)
- **적중 판정** : 화살 위치 ↔ 타겟 중심 거리 < r + 0.25m
- **적중 후 처리** : 페이드 아웃 (0.8초), 제거 후 새 타겟 랜덤 스폰

#### F-07. 점수 및 콤보 시스템
- **기본 점수** : `(거리 점수 + 정확도 보정) * 콤보 배율`
  - 거리 점수 : `5 + |z| * 0.6` (pt)
  - 정확도 보정 : 중심에 가까울수록 최대 +22pt
  - 콤보 배율 : `1 + combo * 0.45` (2콤보부터 적용)
- **콤보 증가** : 연속 적중 시 +1
- **콤보 리셋** : 화살이 타겟 미적중 후 맵 이탈 시
- **HUD 표시** : 좌상단 점수, 중앙 콤보 팝업, 월드 좌표 기반 플로팅 텍스트

### 3.2 보조 기능 (Should Have)

#### F-08. 제스처 디버그 패널 (우측 하단)
- 현재 상태 텍스트 (대기중 / 주먹 인식중 / 시위 당기는 중 N% / 발사 준비)
- 주먹 점수 바 (0~100%, 실시간)
- 손가락 5개 상태 도트 (🔴 굽힘 / 🟢 펼침 / 회색 중간)

#### F-09. 카메라 프리뷰 (우측 하단)
- 웹캠 영상 (좌우 반전, 반투명 50%)
- MediaPipe 스켈레톤 오버레이 (랜드마크 + 커넥터)

#### F-10. 파티클 이펙트
- 적중 시 35개 구체 파티클 폭발
- 색상 : 주황/노랑/빨강/흰색 계열
- 중력 적용, life 600~1200ms, 투명도 fade-out

## 4. 비기능 요구사항

### 4.1 성능
| 항목 | 목표 |
|------|------|
| 렌더링 프레임레이트 | 모바일 30fps 이상 / PC 60fps 이상 |
| MediaPipe 추론 | 카메라 해상도 320×240 (성능 우선) |
| 초기 로드 시간 | CDN 캐시 기준 3초 이내 |
| JS 힙 메모리 | 세션 내 무제한 증가 없음 (dead 오브젝트 즉시 제거) |

### 4.2 호환성
- WebGL 1.0 이상 필수
- WebRTC getUserMedia 지원 필수
- Pointer Lock API : 미지원 시 자유 마우스 폴백

### 4.3 보안 및 프라이버시
- 카메라 스트림은 로컬 처리 전용 (서버 전송 없음)
- MediaPipe 추론 결과(랜드마크 좌표)도 외부 전송 없음
- HTTPS 또는 localhost 환경 필수 (브라우저 카메라 보안 정책)

### 4.4 배포
- 단일 HTML 파일 (인라인 CSS + JS)
- 외부 의존성 : Three.js r128, MediaPipe Hands/Camera/Drawing (CDN)
- 별도 빌드 파이프라인 불필요

## 5. UX / 인터랙션 플로우
```
[랜딩 오버레이]
    조작법 설명 + PLAY 버튼
        ↓ 클릭
[게임 초기화]
    Three.js 씬 생성 → 타겟 5개 스폰 → MediaPipe 카메라 시작
        ↓
[플레이 루프]
    ┌─ 카메라 입력 (자이로 / 마우스) → 조준 방향 갱신
    ├─ 핸드 트래킹 → 제스처 분석
    │     ├─ 주먹 감지 → drawCharge 증가 + 시위 당김 애니메이션
    │     └─ 손 펼침 감지 → shootArrow(drawCharge) 호출
    ├─ 화살 물리 업데이트 (중력, 방향, 트레일)
    ├─ 충돌 감지 → 적중 시 점수/콤보 계산 + 파티클 + 타겟 페이드
    └─ 타겟 스폰 관리 (죽은 타겟 제거 → 새 타겟 스폰)
        ↓
    [반복]
```

## 6. 기술 아키텍처
```
┌─────────────────────────────────────────────────────┐
│                   단일 HTML 파일                     │
├──────────────┬──────────────┬───────────────────────┤
│  Three.js    │  MediaPipe   │  Game Logic (Vanilla JS)│
│  Renderer    │  Hands       │                        │
│  Scene       │  Camera      │  GestureAnalyzer       │
│  Camera      │  Drawing     │  ArrowPhysics          │
│  Bow Mesh    │              │  TargetManager         │
│  Arrow Mesh  │              │  ScoreSystem           │
│  Trail       │              │  HUD Controller        │
│  Particles   │              │  InputRouter           │
└──────────────┴──────────────┴───────────────────────┘
       ↑                ↑
  WebGL Canvas    getUserMedia
                  (Camera Stream)
```

### 6.1 핵심 데이터 흐름
```
Camera Stream
  → MediaPipe Hands (추론, 30fps)
      → onHandResults(landmarks)
          → analyzeHand() → {curled[], ext[], fistScore}
              → hysteresis counter 업데이트
                  → isFist 상태 전환
                      → [주먹] fistStartT 기록
                      → [펼침] shootArrow(drawCharge) 호출
                          → makeArrowMesh() → flyArrows[] push
                              → tick()에서 매 프레임 물리 업데이트
                                  → 충돌 감지 → 점수 처리
```

## 8. 용어 정의

| 용어 | 정의 |
|------|------|
| drawCharge | 0.0~1.0, 시위를 당기는 정도. 발사 속도와 파워에 비례 |
| Fist Score | 5개 손가락 중 굽혀진 손가락 비율 (0.0~1.0) |
| Hysteresis | 오인식 방지용 연속 프레임 카운터. 짧은 노이즈를 상태 전환에서 제외 |
| Trail | 화살 이동 경로를 따라 그려지는 잔상 라인 (25포인트 링 버퍼) |
| Embedded | 타겟 적중 후 화살이 박힌 상태. 2.5초 후 페이드 아웃 |
| Combo | 연속 적중 횟수. 배율 = 1 + combo * 0.45 |
