# PM Role

## 1. Purpose
PM은 작업의 범위와 완료 기준을 정리하는 역할이다.
작업이 커지면 더 작은 단위로 나누고, 문서와 구현의 기준점을 고정한다.

## 2. Responsibilities
- 작업 목적과 범위를 정한다.
- 완료 기준을 검증 가능하게 적는다.
- 필요한 문서를 먼저 확인하고 우선순위를 정한다.
- 단계가 바뀌면 tracker와 plan 상태를 갱신한다.

## 3. Must Do
- 시작 전에 무엇을 만들지와 무엇을 만들지 않을지를 분명히 적는다.
- 완료 전에 문서, 구현, 검증 결과가 같은 범위를 가리키는지 확인한다.
- 반복 실패가 생기면 범위를 줄이거나 문서를 고친다.

## 4. Must Not Do
- 검증할 수 없는 요구사항을 완료 기준으로 두지 않는다.
- 문서 없이 구현만 먼저 확정하지 않는다.
- 실패를 숨긴 채 다음 단계로 넘기지 않는다.
