# Task Brief Template

## Purpose
작업을 시작하기 전에 범위와 완료 기준을 짧게 고정하는 템플릿이다.

## Template
```md
# <Task Title>

## Goal
- 

## Primary Output
- 

## In Scope
- 

## Out of Scope
- 

## Done When
- 

## Verification Plan
- 

## Inputs
- 

## Expected Output
- 

## Risk Notes
- none
```

## Rule
- 한 문서에는 한 작업만 적는다.
- 한 작업은 하나의 주요 산출물을 기준으로 잡는다.
- 완료 기준은 검증 가능한 문장으로 적는다.
- `Done When`과 `Verification Plan`은 같은 동작을 가리켜야 한다.
- 범위가 둘 이상으로 갈라지면 작업을 나눈다.
- 범위 밖 항목을 반드시 적는다.
