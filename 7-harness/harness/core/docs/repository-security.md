# Repository Security

## 1. Purpose
이 문서는 저장소 전체에 공통으로 적용되는 보안 최소 기준을 정의한다.
앱별 문서는 이 기준보다 더 엄격할 수 있지만, 더 느슨할 수는 없다.

## 2. Default Protection Scope
- 인증 정보 (API 키, 토큰, 비밀번호)
- 사용자 개인정보
- 외부 시스템 접근 자격 증명

앱별 security 문서는 보호 대상을 더 구체적으로 적는다.

## 3. Non-Negotiables
- 보호 대상 데이터는 로그, 파일, DB, URL에 남기지 않는다.
- 보호 대상 데이터를 외부 서비스로 전송하지 않는다.
- 인증 정보는 환경 변수 또는 시크릿 매니저로만 관리한다.
- 디버깅 편의를 이유로 예외를 만들지 않는다.
- 보안 규칙이 불분명하면 저장하지 않는 쪽을 기본값으로 둔다.

## 4. Review Rule
- repository-level security는 모든 앱의 기본선이다.
- app-level security는 도메인에 맞는 추가 규칙을 정의한다.
- 충돌 시 더 엄격한 규칙을 따른다.

## 5. Verification
- 금지 패턴은 자동 검사할 수 있어야 한다.
- 자동 검사로 못 잡는 데이터 흐름은 리뷰 단계에서 확인한다.
- 보안 의심 사항이 남으면 기능 완료로 처리하지 않는다.
