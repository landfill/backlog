# Ongoing Work

## 목적

진행 중인 작업의 task brief, ongoing plan, review report를 함께 관리한다.

## 파일 규칙

- `YYYY-MM-DD-<slug>-task-brief.md`
- `YYYY-MM-DD-<slug>-ongoing-plan.md`
- `YYYY-MM-DD-<slug>-review-report-<stage>.md`

## 운영 규칙

- 같은 작업의 task brief와 ongoing plan은 같은 slug를 쓴다
- 상태가 바뀌면 current owner가 ongoing plan을 바로 갱신한다
- handoff가 생기면 review report를 추가하고 cleanup 상태를 기록한다
