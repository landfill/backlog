# Teams 연동 설계

## 역할

Teams는 **실시간 커뮤니케이션 채널**이다.
사용자 개입 요청, 처리 진행 알림, 긴급 에스컬레이션에 사용한다.

## 사용 에이전트별 권한

| 에이전트 | 읽기 | 쓰기 | 주요 용도 |
|---|---|---|---|
| Executor | ✅ | ✅ | 알림 발송, 사용자 응답 수신 |
| Lead | ✅ | ❌ | 사용자 응답 확인 |

다른 에이전트는 Teams를 직접 사용하지 않는다. 필요 시 Executor를 통해 발송.

---

## 메시지 유형

### 1. 사용자 개입 요청

`human-in-the-loop.md`에서 정의한 형식으로 발송.

**발송 대상**: 개입 요청 대상자 (티켓 보고자, 팀 리더 등)
**방식**: 1:1 DM 또는 팀 채널

```markdown
🔔 [JIRA-1234] 확인 요청

**상황**: 티켓 분류가 불명확합니다
**필요한 판단**: 운영성 티켓인지, 프로젝트성 티켓인지 확인

선택지:
1. 운영 — 즉시 매뉴얼 기반 처리
2. 프로젝트 — 일정 수립 후 단계별 진행
3. 직접 지시

⏰ 응답 기한: 14:00 (미응답 시 운영으로 분류하여 처리)
```

### 2. 처리 진행 알림

**발송 시점:**
- 처리 시작
- 중간 진행 (프로젝트성 마일스톤 완료 등)
- 처리 완료
- 지연 예상

```markdown
✅ [JIRA-1234] 처리 완료

runbook 4.1절 절차 적용, 서비스 정상 확인.
상세: [Jira 티켓 링크]
```

### 3. 긴급 에스컬레이션

**발송 조건**: 보안 이슈, 장애 확산 위험 등 즉각 대응 필요

```markdown
🚨 [JIRA-1234] 긴급 — 확인 필요

**상황**: Gate 2에서 보안 위반 의심 감지
**영향 범위**: [시스템/서비스 명시]
**에이전트 행동**: 처리 중단 상태, 사람 확인 대기 중

즉시 확인 부탁드립니다.
```

### 4. 리마인더

타임아웃 50% 시점에 발송 (`human-in-the-loop.md` 참조).

### 5. 보고서 링크 공유

일별/주별 보고서 생성 후 Confluence 링크 공유.

```markdown
📊 주별 보고서 (2026-W13)

처리 건수: 15건 (자율 12건, 개입 3건)
[Confluence 보고서 링크]
```

## 채널 전략

| 메시지 유형 | 채널 | 이유 |
|---|---|---|
| 개입 요청 (개인) | 1:1 DM | 특정인의 판단 필요 |
| 개입 요청 (팀) | 팀 채널 | 누구든 응답 가능한 질문 |
| 처리 알림 | 팀 채널 | 팀 전체 가시성 |
| 긴급 에스컬레이션 | 1:1 DM + 팀 채널 | 즉각 주목 필요 |
| 보고서 공유 | 팀 채널 | 정기 공유 |

**채널 설정:**

```yaml
channels:
  team_channel: ""        # 팀 전체 채널 ID
  escalation_channel: ""  # 에스컬레이션 전용 채널 (선택)
```

> ⚠️ 실제 채널 ID는 Phase 4에서 구체화한다.

## 응답 수신 처리

### Adaptive Card 기반 응답

선택지가 있는 개입 요청은 Adaptive Card로 발송하여 버튼 클릭으로 응답 가능하게 한다.

**응답 수신 플로우:**

```
[Executor] Adaptive Card 발송
  ↓
사용자 버튼 클릭 (또는 텍스트 응답)
  ↓
[Executor] 응답 수신 + Lead에 전달
  ↓
[Lead] 응답 해석 + 파이프라인 재개
  ↓
[Reporter] 의사결정 로그에 기록
```

### 텍스트 응답 처리

사용자가 선택지 대신 자유 텍스트로 응답한 경우:
- Lead가 텍스트를 해석하여 행동 결정
- 해석이 모호하면 재확인 요청 (1회 한정)

---

## 메시지 형식 규칙

- 모든 메시지에 `[🤖 팀원에이전트]` 식별자 포함
- 티켓 번호를 항상 포함하여 추적 가능하게
- 한 메시지에 하나의 주제만 (복수 건 묶지 않음)
- 긴급도에 따른 아이콘: 🔔 일반, ✅ 완료, ⏰ 리마인더, 🚨 긴급
- 메시지 길이: 최대 500자 (상세는 링크로 제공)

## 에러 처리

| 에러 | 재시도 | 대응 |
|---|---|---|
| 401 Unauthorized | ❌ | 토큰 만료 → Lead 보고 |
| 403 Forbidden | ❌ | 채널 접근 불가 → Lead 보고 |
| 429 Throttled | ✅ 3회 | Retry-After 준수 |
| 발송 실패 | ✅ 2회 | 실패 시 Outlook 메일로 폴백 |

### Teams 장애 시 폴백

Teams가 불가용하면:
1. Outlook 메일로 동일 내용 발송
2. Jira 댓글로 기록
3. 의사결정 로그에 "Teams 장애, Outlook 폴백" 기록
