# 보고 생성 규칙 (Reporting)

## 목적

에이전트 팀의 처리 현황과 판단 과정을 **팀 리더와 유관자가 파악할 수 있도록** 정기적으로 보고서를 생성한다.
보고서는 의사결정 로그를 집계한 산출물이다.

## 담당 에이전트

- **보고서 작성**: Reporter
- **보고서 품질 검증**: Guardian (Gate 3)
- **보고서 발행 (Confluence/Teams)**: Executor
- **보고 주기 감독**: Lead

## 내부 handoff 보고와 외부 보고의 구분

- review report는 단계 간 handoff용 내부 산출물이다
- 일별/주별/유관자/프로젝트 상태 보고는 사람에게 공유하는 외부 보고다
- review report 형식은 repository-level `harness/core/templates/review-report-template.md`를 따른다
- 외부 보고가 생성되더라도 review report를 생략하지 않는다

## 보고 유형

| 유형 | 주기 | 대상 | 생성 트리거 |
|---|---|---|---|
| 일별 보고 | 매일 (업무 종료 시) | 팀 리더 | 스케줄 (매일 17:00) |
| 주별 보고 | 매주 (금요일) | 팀 리더 | 스케줄 (금요일 17:00) |
| 유관자 공유 | 수시 | 유관자 | 티켓 처리 완료 또는 주요 진행 시 |
| 프로젝트 상태 | Coordinator 설정 주기 | 팀 리더 + 유관자 | Coordinator 트리거 |

## 일별 보고

### 포함 항목

```markdown
# 일별 보고 — YYYY-MM-DD

## 요약
- 처리 건수: N건 (자율 N건 / 개입 N건)
- 신규 배정: N건
- 미완료: N건
- 에스컬레이션: N건

## 처리 내역

| 티켓 | 분류 | 확신도 | 처리 결과 | 소요 시간 | 비고 |
|---|---|---|---|---|---|
| JIRA-1234 | 운영 | 높음 | 해결 | 45분 | |
| JIRA-1235 | 프로젝트 | 중간 | 진행중 | — | 1단계 완료 |

## 사용자 개입 건

| 티켓 | 개입 사유 | 상태 | 비고 |
|---|---|---|---|
| JIRA-1236 | 분류 불명확 | 응답 대기 | 12:00 요청 발송 |

## 특이사항
- (새 패턴 발견, 매뉴얼 갭, Guardian 반려 등)
```

## 유관자 공유

티켓 처리 완료 또는 주요 진행 시점에 해당 유관자에게 공유.
Confluence 페이지가 아닌 **Teams 메시지 또는 Jira 댓글**로 전달 (간결함 우선).

### 포함 항목

- 티켓 번호 + 제목
- 처리 결과 요약 (1~2줄)
- 판단 근거 요약 (어떤 매뉴얼/사례 기반)
- 의사결정 로그 링크 (상세 필요 시)

## 프로젝트 상태 보고

Coordinator가 관리하는 프로젝트성 티켓의 진행 보고.
일반 보고와 별도 주기로 생성 (Coordinator가 결정).

### 포함 항목

```markdown
# 프로젝트 상태 보고 — JIRA-XXXX

## 개요
- 프로젝트명: [프로젝트/티켓 제목]
- 시작일: YYYY-MM-DD
- 예상 완료일: YYYY-MM-DD
- 현재 단계: [마일스톤 N]

## 마일스톤 현황

| 마일스톤 | 상태 | 예정일 | 완료일 | 비고 |
|---|---|---|---|---|
| MS1: 요구분석 | ✅ 완료 | MM-DD | MM-DD | |
| MS2: 개발 | 🔄 진행중 | MM-DD | — | 70% |
| MS3: 테스트 | 📋 예정 | MM-DD | — | |

## 리스크 / 이슈

| 유형 | 내용 | 영향도 | 대응 |
|---|---|---|---|
| 리스크 | (설명) | 높음/중간/낮음 | (대응 계획) |

## 다음 주 계획
- [예정 작업 목록]
```

## 보고서 생성 → 발행 플로우

```
[스케줄 트리거] 보고 시간 도달
  ↓
[Reporter] 의사결정 로그 집계
  ↓
[Reporter] 보고서 템플릿에 맞춰 초안 작성
  ↓
[Guardian] Gate 3: 보고 품질 검증
  │
  ├─ 반려 → Reporter 보완
  │
  └─ 통과
      ↓
  [Executor] Confluence 페이지 생성
      ↓
  [Executor] Teams 채널에 링크 공유
      ↓
  [Executor] Outlook 메일 발송 (주별 보고만)
```

## 보고 미생성 시 대응

보고 주기를 건너뛰지 않는다 (AGENTS.md 제약).

| 상황 | 대응 |
|---|---|
| 처리 건수 0건 | "금일 처리 건 없음" 보고 생성 (빈 보고도 보고) |
| 시스템 장애로 생성 불가 | Lead에 즉시 알림 + 복구 후 지연 보고 생성 |
| Guardian 반려로 지연 | 보완 후 최대한 빨리 발행 + 지연 사유 명시 |
