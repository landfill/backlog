# 도메인 용어 사전 (Glossary)

## 목적

에이전트 팀이 사용하는 용어의 정의를 통일한다.
티켓 분석, 매뉴얼 정제, 보고서 작성 시 이 사전을 참조한다.

---

## 시스템 용어

> ⚠️ 실제 시스템/서비스 용어는 운영 시 구체화한다.

| 용어 | 정의 | 비고 |
|---|---|---|
| (운영 시 추가) | | |

---

## 에이전트 팀 용어

| 용어 | 정의 |
|---|---|
| **Lead** | 에이전트 팀의 오케스트레이터. 분류, 계획, 위임, 최종 승인 담당 |
| **Coordinator** | 프로젝트성 티켓에서 Lead가 스폰하는 조율자. 일정/마일스톤 관리 |
| **Analyst** | 자료 조사 + 정리를 수행하는 분석가 |
| **Executor** | 확정된 솔루션을 외부 시스템에서 실행하는 실행자 |
| **Reporter** | 의사결정 로그, 보고서, 템플릿을 관리하는 보고관 |
| **Guardian** | 파이프라인 각 단계의 품질 게이트를 운영하는 검수관 |

---

## 프로세스 용어

| 용어 | 정의 |
|---|---|
| **DAG** | Directed Acyclic Graph. Lead가 생성하는 작업 의존성 그래프 |
| **Gate** | Guardian이 운영하는 품질 검증 관문. Gate 1(분석), Gate 2(실행+보안), Gate 3(보고) |
| **확신도** | Lead가 솔루션 선택 시 부여하는 판단 확실성. 높음/중간/낮음 |
| **Human-in-the-Loop** | 에이전트가 사람에게 판단을 요청하는 프로토콜 |
| **에스컬레이션** | 에이전트가 처리할 수 없어 사람에게 넘기는 행위 |
| **Runbook** | 정제된 운영 매뉴얼. 티켓 유형별/시스템별로 분류 |
| **Past Solution** | 과거 티켓 해결 사례를 패턴화한 문서 |
| **Intake Log** | 원본 자료 수집 이력 기록 |
| **신뢰도** | Past Solution의 검증 정도. 초기/중간/높음/낮음 |

---

## 티켓 분류 용어

| 용어 | 정의 |
|---|---|
| **운영 티켓** | 즉각적인 대응이 필요한 티켓. 장애, 버그, 서비스 요청 등 |
| **프로젝트 티켓** | 계획적 진행이 필요한 티켓. 기능 개발, 마이그레이션 등 |
| **불명확 티켓** | 운영/프로젝트 분류가 모호한 티켓. 사용자 확인 필요 |

---

## 보고서 용어

| 용어 | 정의 |
|---|---|
| **의사결정 로그** | 티켓 처리 건별로 판단 근거를 기록한 마크다운 파일 |
| **일별 보고** | 하루 처리 현황을 팀 리더에게 보고하는 문서 |
| **주별 보고** | 주간 처리 현황 + 지표를 팀 리더에게 보고하는 문서 |
| **유관자 공유** | 특정 티켓/프로젝트의 진행을 관련자에게 공유하는 문서 |

---

## 용어 추가 규칙

- 새로운 도메인 용어가 발견되면 Reporter가 이 문서에 추가를 제안한다
- 정제 과정에서 원본에 등장하는 약어/전문용어를 이 사전에 등록한다
- 용어 정의가 모호하면 `[정의 확인 필요]` 태그를 남기고 사용자에게 확인 요청한다
