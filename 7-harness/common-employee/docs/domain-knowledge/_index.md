# 도메인 지식 베이스 (Domain Knowledge)

## 구조

```
domain-knowledge/
├── raw/                    # 원본 자료 (정제 전)
│   └── _intake-log.md      # 수집 이력
├── runbooks/               # 정제된 운영 매뉴얼
│   ├── by-ticket-type/     # 티켓 유형별 처리 절차
│   └── by-system/          # 시스템별 처리 절차
├── past-solutions/         # 과거 해결 사례 (패턴화)
├── escalation-matrix.md    # 에스컬레이션 경로/기준
└── glossary.md             # 도메인 용어 사전
```

## 규칙

### 원본 자료 (raw/)
- 위키, PPT, Word 등 원본을 변환하여 저장
- 반드시 `_intake-log.md`에 수집 일시, 출처, 상태를 기록
- 원본은 참조용이며, 에이전트가 직접 참조하지 않음

### 정제된 매뉴얼 (runbooks/)
- 에이전트가 실제로 참조하는 문서
- 구조: 제목 → 적용 조건 → 절차 → 예외 사항 → 출처
- 매뉴얼이 현실과 다른 경우 업데이트 제안 플래그를 남김

### 과거 사례 (past-solutions/)
- 실제 티켓을 해결한 사례를 패턴화
- 구조: 원본 티켓 → 증상 → 원인 → 해결 방법 → 재사용 횟수
- 재사용될수록 신뢰도 상승

### 상태 관리
- 각 문서에 `last_verified` 날짜를 기록
- 3개월 이상 검증되지 않은 문서는 재검증 대상으로 표시
